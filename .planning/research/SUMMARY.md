# Research Summary: CanaryOS Scam Text Classifier

**Domain:** On-device mobile scam / phishing text classification
**Researched:** 2026-04-01
**Overall confidence:** MEDIUM

---

## Executive Summary

The existing MobileBERT model trained on the UCI SMS Spam corpus is broken for a known reason: the training distribution is too narrow. Public SMS spam datasets top out at ~6K samples and are overwhelmingly legacy prize/lottery SMS from the 2010s. Modern scam vectors — crypto/pig-butchering, romance grooming, government impersonation, package delivery smishing, and tech support — are structurally absent. The model learned surface-level vocabulary patterns rather than underlying persuasion mechanics, making it brittle against any novel phrasing.

The path to a working classifier requires three things in strict order: a synthetic dataset covering all 8 active scam vectors (generated by Gemini 2.5 Flash via the google-genai SDK, filtered for label quality), a student model architecture that fits inside the 50MB INT8 budget (TinyBERT-4 at ~3.6MB INT8 is the best candidate), and knowledge distillation from a large teacher (DeBERTa-v3-large, fine-tuned server-side) to close the accuracy gap before quantization. These are dependent steps — distillation requires good data and a trained teacher; the teacher must be fine-tuned before student training; data must be validated before any training starts.

The 2024–2025 research landscape confirms that TinyBERT-4 intermediate-layer distillation (attention matrix + hidden state transfer, not just soft labels) achieves near-BERT accuracy at 7.5x compression and 9.4x speedup. QAT is non-negotiable for BERT-family models — PTQ degrades GLUE-class accuracy significantly, while QAT recovers within 1% of FP32 baseline. The multi-label intent head (urgency, authority, financial request, remote access, reward/lottery, impersonation, romance, crypto) should be added after the binary classifier is stable, as a second training pass, not from scratch.

The field has also moved: ModernBERT (Dec 2024) is now SOTA for encoder classification but its 149M parameter base size and absence of a TFLite conversion path make it unsuitable for this milestone. It is a candidate for Phase 2 when the model budget may relax and CoreML becomes an option.

---

## Key Findings

**Stack:** TinyBERT-4 student, DeBERTa-v3-large teacher, TF Model Optimization Toolkit for QAT, HuggingFace Transformers for training — all server-side; TFLite INT8 for device deployment.

**Architecture:** Single shared encoder with two heads: binary sigmoid (scam/safe) + 8-label multi-label sigmoid for intent vectors. One forward pass, one model file.

**Critical pitfall:** Training on synthetic data without quality filtering produces label noise that causes the model to learn generator artifacts rather than scam patterns. Self-consistency filtering (run teacher zero-shot on generated samples, discard mismatches) is required before any training.

## Implications for Roadmap

Based on research, suggested phase structure for milestone v1.0:

1. **Synthetic Dataset Generation** — gates everything downstream
   - Addresses: TEXT-01 (synthetic scam/safe dataset)
   - Avoids: Pitfall of narrow training distribution that broke the current model
   - Deliverable: 16K–24K labeled samples across 8 vectors + safe class, with hard negatives

2. **Architecture Benchmark + Binary Baseline** — validates model selection before investing in distillation
   - Addresses: TEXT-02 (architecture benchmark)
   - Avoids: Committing to DistilBERT (over budget) or MobileBERT (larger than TinyBERT-4 for same accuracy)
   - Deliverable: F1 comparison table, selected student architecture

3. **Teacher Fine-Tuning (DeBERTa-v3-large)** — prerequisite for distillation
   - Addresses: prerequisite for TEXT-04
   - Note: Server-side only, never deploys to device
   - Deliverable: Fine-tuned teacher checkpoint, F1 > 0.95 on test set

4. **Knowledge Distillation to Student** — main accuracy improvement step
   - Addresses: TEXT-04 (knowledge distillation)
   - Avoids: Soft-labels-only distillation (intermediate layer transfer is strictly better, confirmed 2024)
   - Expected gain: 3–5 F1 points over direct fine-tune of student

5. **Multi-Label Intent Head** — differentiator, builds on stable binary classifier
   - Addresses: TEXT-03 (intent classifiers for scam sub-types)
   - Avoids: 8 separate models (memory/latency prohibitive)
   - Deliverable: Single model with 8 sigmoid intent heads

6. **Int8 QAT + TFLite Export** — production step
   - Addresses: TEXT-05 (QAT), TEXT-06 (deployed model)
   - Avoids: PTQ (proven accuracy degradation on BERT family)
   - Deliverable: INT8 TFLite model ≤ 10MB, validated in canaryapp

**Phase ordering rationale:**
- Data generation must precede all training steps — no training without a validated dataset
- Binary baseline must precede teacher fine-tuning — confirms the task is learnable at all
- Teacher must precede distillation — distillation is undefined without a trained teacher
- Binary classifier must be stable before adding multi-label head — avoids training instability
- QAT is the last step — simulating quantization during training requires final architecture to be fixed

**Research flags for phases:**
- Phase 3 (teacher fine-tune): likely needs deeper investigation — DeBERTa-v3-large requires significant GPU; confirm training environment (Colab A100 vs local) before starting
- Phase 4 (distillation): needs phase-specific research on layer mapping (TinyBERT-4 has 4 layers, teacher has 24 — mapping strategy affects final accuracy)
- Phase 5 (multi-label): threshold tuning per label is non-trivial given class imbalance across vectors

## Gaps to Address

- Exact layer mapping strategy for TinyBERT-4 (4 layers) to DeBERTa-v3-large (24 layers) distillation needs phase-specific research before implementation
- Training infrastructure decision needed: DeBERTa-v3-large fine-tuning requires >16GB GPU VRAM — confirm Colab A100, Lambda Labs, or similar before roadmap scheduling
- ModernBERT on-device feasibility should be revisited in Phase 2 — no TFLite path exists today but the model is significantly faster than MobileBERT at the same task
- Calibration strategy (temperature scaling vs Platt scaling) for multi-label intent heads needs evaluation once heads are trained
