<objective>
Apply the JS divergence gate, two-pass quality filter, stratified train/val/test split, and 100-sample human review to produce the final dataset files consumed by Phase 2.

Purpose: Raw synthetic data must pass quality gates before any model training. The JS divergence check (D-08) catches mode collapse.   <action>
Create `research/scripts/filter_and_split.py` that implements the full quality pipeline. The script runs sequentially through 6 stages, halting on any gate failure.

**Stage 0: Pre-checks**

- Assert `research/data/synthetic_raw.jsonl` exists (exit 1 with "synthetic_raw.jsonl not found — Plan 02 must complete first" if missing)
- Assert `research/data/holdout_realworld.jsonl` exists (needed for BART baseline)
- Pre-download bart-large-mnli model with disk space check: `from transformers import AutoModelForSequenceClassification, AutoTokenizer; AutoModelForSequenceClassification.from_pretrained("facebook/bart-large-mnli"); AutoTokenizer.from_pretrained("facebook/bart-large-mnli")`. If download fails, exit with descriptive error about disk space or network.

**Stage 1: BART Accuracy Baseline on Holdout (addresses review item 5)**

Before using bart-large-mnli as a quality filter, validate its accuracy on known-labeled holdout data:
- Load `research/data/holdout_realworld.jsonl` (known labels)
- Run bart-large-mnli zero-shot classification (`candidate_labels=["scam", "safe"]`) on 100 random holdout samples (or all if fewer than 100)
- Compute accuracy: `correct / total`
- Print results: `"BART baseline accuracy on holdout: {accuracy:.2%} ({correct}/{total})"`
- If accuracy < 0.70: exit 1 with `"BART accuracy on holdout is {accuracy:.2%} — too low to use as quality filter. Consider using a different zero-shot model or lowering the self-consistency threshold."`
- If accuracy >= 0.70 but < 0.85: print warning `"BART accuracy is {accuracy:.2%} — moderate. Self-consistency filter may be noisy. Consider lowering threshold to 0.5."`
- If accuracy >= 0.85: print `"BART accuracy is {accuracy:.2%} — good. Proceeding with default threshold 0.6."`
- Set `effective_default_threshold` based on BART accuracy: 0.6 if accuracy >= 0.85, 0.5 if accuracy >= 0.70

**Stage 2: JS Divergence Gate (D-08, mandatory pre-filter)**

- Load `research/data/synthetic_raw.jsonl`
- Compute token-unigram Jensen-Shannon divergence for all scam vector pairs using `scipy.spatial.distance.jensenshannon`
- Implementation: for each vector, collect all texts, lowercase, split on whitespace to get tokens, build frequency distribution. Normalize to probability. Compute JSD for every pair of vectors.
- Print divergence matrix with interpretation guidance (addresses review item 12):
  ```
  JSD Matrix (interpretation: <0.05 = likely mode collapse; 0.1-0.2 = expected for related types; >0.3 = strong diversity):
  crypto_investment vs romance_grooming: 0.187
  crypto_investment vs tech_support: 0.231
  ...
  ```
- Gate: if ANY pair has JSD < 0.05, halt with error message naming the collapsed pair and suggesting: "Revise prompt templates for these vectors to increase vocabulary diversity. Re-run Plan 02." Do NOT proceed to filtering.
- On pass: save full matrix to `research/data/jsd_matrix.json`

**Stage 3: LLM Self-Consistency Check (Pass 1 of two-pass filter)**

- Use `transformers.pipeline("zero-shot-classification", model="facebook/bart-large-mnli", device=-1)` (CPU)
- For each sample, run zero-shot classification with candidate_labels `["scam", "safe"]`
- Keep sample if predicted label matches `sample["label"]` AND confidence >= `effective_default_threshold` (from Stage 1)
- **Per-vector discard logging (addresses review concern about adaptive threshold):** Track discards per vector. Print running stats every 2000 samples: `"[12000/27000] crypto_investment: 85% kept, romance_grooming: 72% kept..."`
- If any vector loses >40% of samples at current threshold:
  - **Before lowering threshold:** Log 20 random discarded samples for that vector with their BART confidence scores. Print: `"[REVIEW] romance_grooming: 20 discarded samples before threshold change:"`
  - Lower threshold to max(0.5, effective_default_threshold - 0.1) for that vector only
  - Print: `"Adaptive threshold: romance_grooming lowered from {old} to {new} (discard rate was {rate:.1%})"`
- Track total kept vs discarded

**Stage 4: Rule-Based Post-Filter (Pass 2 of two-pass filter)**

- Remove exact duplicate `text` values (after whitespace stripping) using `pandas.DataFrame.drop_duplicates(subset=["text"])`
- Remove samples where `text.split()` has fewer than 15 tokens
- Log: duplicates removed count, sub-15-token removed count

**Stage 5: Stratified Split**

- Use `sklearn.model_selection.train_test_split` with `stratify` on `vector` field, `random_state=42`
- First split: 90% train+val, 10% test
- Second split on train+val: ~88.9% train, ~11.1% val (yields ~80/10/10 of original total)
- Add `"split"` field to each sample: `"train"`, `"val"`, or `"test"`
- Write train+val samples to `research/data/synthetic_scam_v1.jsonl` (train and val together, distinguished by split field)
- Write test samples to `research/data/test_split.jsonl`
- Print: `"Note: splits use random_state=42 for deterministic reproducibility across phases"`

**Stage 6: Summary report**

- Total raw samples loaded
- BART baseline accuracy on holdout
- JSD gate: pass/fail + min JSD pair + interpretation
- Self-consistency filter: kept/discarded counts per vector, effective threshold per vector, any adaptive changes made
- Rule-based filter: duplicates removed, sub-15-token removed
- Final counts: train / val / test per vector
- Safe class composition: hard negative breakdown
- Post-filter total vs pre-filter total (yield percentage)

**Review process (TEXT-01 success criterion 4) with quantitative acceptance criteria (addresses review item 9):**

1. Open `research/data/human_review_sample.jsonl` and read each of the 100 samples.

2. **Quantitative rejection criteria — REJECT the dataset if any of these are true:**
   - **Mode collapse:** >30% of any single vector's samples in the 100-sample set share identical surface phrases or sentence structures (e.g., all crypto scams open with "Dear investor" or all use "guaranteed returns")
   - **Topical over-specificity:** Any vector covers only 1-2 sub-scenarios (e.g., all tech_support = "Microsoft Windows virus" with no "Apple", "antivirus subscription", or "router" variants)
   - **Mislabeling rate:** >5% of the 100 samples are clearly mislabeled (scam labeled as safe, or vice versa)
   - **Synthetic hard negatives:** Hard-negative safe samples read as obviously computer-generated (e.g., "Dear customer, your verification code is 123456. Thank you for banking with us." — too generic to be useful)
   - **Length deficit:** >15% of samples are only 1-2 generic sentences with no specificity

3. **What to look for per vector (spot check at least 2-3 samples per vector):**
   - crypto_investment: Does it vary between pig butchering, fake exchanges, and crypto ATM scams?
